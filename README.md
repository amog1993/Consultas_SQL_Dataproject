# Consultas_SQL_Dataproject

🎬 Proyecto SQL – Análisis de Datos con la Base de Datos Sakila

📌 Descripción del Proyecto

Este proyecto consiste en la resolución de una serie de ejercicios prácticos utilizando la base de datos Sakila, con el objetivo de aplicar conceptos fundamentales e intermedios de SQL en un contexto de análisis de datos.

A través de distintas consultas, se respondieron preguntas relacionadas con clientes, alquileres, películas y categorías, simulando escenarios típicos de análisis de negocio.

🎯 Objetivos del Proyecto

Aplicar consultas SQL sobre una base de datos relacional en un entorno profesional.

Demostrar el manejo de la herramienta DBeaver para la gestión y ejecución de consultas.

Realizar consultas sobre una sola tabla, aplicando filtros y funciones básicas.

Trabajar con relaciones entre múltiples tablas mediante el uso de JOIN.

Implementar subconsultas para resolver problemas de mayor complejidad.

Crear y utilizar vistas (VIEW) para estructurar consultas reutilizables.

Utilizar estructuras de datos temporales cuando la lógica del análisis lo requirió.

Aplicar buenas prácticas en la escritura y optimización de consultas SQL.

🧠 Competencias Técnicas Demostradas

Durante el desarrollo del proyecto se aplicaron los siguientes conocimientos:

🔹 Manejo de DBeaver

Conexión a la base de datos.

Ejecución y organización de consultas.

Gestión de scripts SQL.

🔹 Consultas sobre una sola tabla

Uso de SELECT, WHERE, ORDER BY.

Aplicación de funciones de agregación (COUNT, SUM, etc.).

Filtrado de datos con condiciones específicas.

🔹 Relaciones entre tablas

Uso de INNER JOIN.

Comprensión de claves primarias y foráneas.

Construcción de consultas con múltiples relaciones.

🔹 Subconsultas

Uso de subconsultas en cláusulas SELECT, WHERE y HAVING.

Resolución de problemas analíticos mediante consultas anidadas.

🔹 Vistas

Creación de vistas para simplificar consultas complejas.

Uso de vistas como base para análisis posteriores.

🔹 Estructuras temporales

Uso de estructuras temporales para organizar resultados intermedios.

Separación lógica de consultas complejas en pasos manejables.

🔹 Buenas prácticas

Uso consistente de alias.

Agrupación correcta de columnas en GROUP BY.

Eliminación de JOIN innecesarios para optimización.

Escritura clara y estructurada del código SQL.

🛠️ Tecnologías Utilizadas

DBeaver
PostgreSQL

Base de datos: Sakila

🧠 Principales Aprendizajes

Importancia de agrupar correctamente todas las columnas no agregadas.

Comprensión de cómo las relaciones entre tablas influyen en la lógica de consulta.

Optimización de consultas eliminando JOIN innecesarios.

Traducción de preguntas de negocio a consultas SQL estructuradas.
